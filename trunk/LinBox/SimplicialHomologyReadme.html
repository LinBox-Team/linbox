See the Homology package home for a description of the capabilities to compute homology of simplicial complexes and to compute Smith Normal Forms of integer matrices, as well as information on download and installation. Also, the manual (34 pgs) of the homology package is recommended as guide to the construction of commands for this form.

This form allows you to use the GAP Homology package installed at the University of Delaware as an alternative to downloading and installing it locally. It is a beta release. AS SUCH, THE SERVER MAY BE DOWN FOR BRIEF PERIODS OF TIME, FOR WHICH WE APOLOGIZE. We welcome suggestions and problem reports. About the server the primary contact is Brian Gold. About the homology package the contact is David Saunders (or other package authors).

Instructions for using this Gap Interface:
       1.   Choose one of the following options:
             a. Enter the URL of a file containing Gap commands in the first box, or
             b. Enter a list of Gap commands into the second box, each command on its own line.

       2.   Choose an option pertaining to the level of comments generated by Gap.

       3.   Choose an option pertaining to the timing generated by Gap.

      4.   Choose an option pertaining to how you want to receive the results.

An example of a list of Gap Commands (and their explanations):

RequirePackage("homology");                             # Load the necessary library to perform the calculations
cx := SCGeneralizedKleinBottle(4);                    # cx <- a GeneralizedKleinBottle complex.
SimplicialBoundaryMaps(cx, 4, "kb4-3");           # Store the 4th boundary map in local file "kb4-3".
SMSmithForm("kb4-3");                                    # Compute Smith Normal Form of the matrix in file "kb4-3".
SimplicialHomology(cx);                                     # Generate simplicial homology of the complex in GAP variable 'cx'.

Helpful Hints:

To suppress the GAP output of a statement, end your statement in ';;' instead of ';'. This is useful for example, when you are storing a large matrix into a local variable and you do not want to see the matrix redisplayed as GAP output (which would occur if you end the statement with ';').

Common Errors:

Leaving off the semicolon ';' at the end of every gap command
Not using ':=' to store into a local variable (probably only using '=', which is an error)
Not surrounding a filename of a matrix with " " (ie "yourmatrix")
