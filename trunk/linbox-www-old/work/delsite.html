Summary of computing resources available at Delaware
for the Spaghetto team - Linbox project.

  1. The group algebra for file access.

    Team members are all in the "algebra" group at Delaware.

  2. The directory /usr/local/algebra/.

    This directory contains most files and directories of interest for computer algebra. Directories or links are there for Maple, Mathematica, Saclib, etc. In particular /usr/local/algebra/linbox/ holds project related stuff.

    These directories belong to the algebra group. Algebra group members can write them. Please be careful not to inadvertently remove something there. Also please follow the convention of making everything you put there at least readable/executable to algebra group members. As much as possible please also make things writable by algebra group.

    A good access policy is chmod g+u [file] Thus make your usage compatible with umask 002 or 007.

    You can change the group of a file with chgrp algebra [file].

  3. Local organization and administration.

    A summary about the local setup is at the at EE/CIS FAQ page . Mike Davis is the chief systems administrator for the eecis network, a shared resource of the Computer and Information Science and Electrical Engineering departments. Problem reports and system queries can be sent to the newsgroup system@eecis.udel.edu for resolution. External users please copy saunders@eecis.udel.edu when writing to system so there is a local team member aware of the issue.

  4. High Performance Cluster.

    These 16 machines each have 4 processors, 512Mb, and are connected by fast ethernet and Myrinet. They do not share files with the rest of the eecis network. Each machine has two names. One of the names is a number prefixed by nsfri, thus nsfri1 nsfri2 .. nsfri16. The second name is the name of a volcano, eg. maui. You can use either one. If you find you do not have a home directory when you log on there, contact saunders to remedy that.

    Please use the cluster as a compute/development server only (avoid email, latex, netscape, ...). For long and/or timed computations, use the reservation system. For more info about the cluster see
    http://www.eecis.udel.edu/nsfri/.

  5. Other machines.

    ren and taygeta, for example, have the same architecture as the cluster and are available for general purposes.

    The local command sysinfo [machine] will give some specs on the machine. Don't give sysinfo without an argument. You will get a very long output!

  6. Linbox project web page.

    http://www.eecis.udel.edu/~caviness/linbox/

    For now this is a private web page for team communications. The directory is writable by group algebra, so you can add/modify things on the web page. Please do.
