Equation Editor for GNOME

Customer: The free software (GNOME) community

Rationale:

  • Cost: Typsetting and displaying mathematical text is a very complex process that involves deep knowledge of the underlying theory. We can take code from existing sources (e.g., LaTeX and Mozilla), but there may be licensing issues that complicate that.
  • Benefit: The GNOME productivity applications need an interface for high-quality mathematical typesetting from within a word processor, licensed unrestrictively.

Goal: Create an embeddable, interactive interface for mathematical typesetting, display, and editing. This interface could work standalone or equivalently be accessible from a Bonobo container, e.g. a word processor.

Scope: We plan to abstract from some existing source the code to typeset and render the mathematical text, so as to make compliance with existing standards easier. So, display and typesetting should be relatively feature-complete assuming we do not run into problems with licensing. We will also implement the functionality for user input as well as a GUI to allow the program to run standalone. Persistence (i.e., saving to disk) will also be implemented, using Bonobo to permit embedding as a component in other documents. We will not implement import/export functionality at this time.

Through the use of the Bonobo component model, we will obtain many capabilities for free. For example, embedding a renderer using this program inside a symbolic manipulation package is a natural consequence of the component system. However, provided that we have enough time, we will develop a CORBA-based API to allow other programs and scripts to interface with the equation editor. This will provide a wealth of new capabilities. We do not intend to implement any form of symbolic, algebraic, or numerical manipulation or computation.

System Functions:

  • Typesetting and rendering mathematical text
  • (Corollary to above) Rendering mathematical symbols
  • GUI-based user input of mathematical expressions
  • Save/load to/from persistent stream (implements BonoboPersistStream interface)
  • Multiple undo and redo
  • Embed other objects into an equation (i.e., act as a container and a containee)
  • Autosave in standalone mode
  • Create an equation from a script or an external program
  • Cut and paste (i.e. X selection support)
  • Change presentation semantics, e.g. from inline to block

System Attributes:

  • Embeddable as a Bonobo component
  • Acts as a Bonobo container to embed other things
  • Runnable as a standalone program
  • User-friendly (i.e., graphical toolbars, etc.), but suitable for "power users" (i.e., hotkeys, etc.)
  • Fast, flicker-free reflow
  • Robustness

Use Cases:

  • Running standalone
  • Use as a rendering engine in symbolic manipulator
  • Embedding into word processor documents
  • Multiple level undo and redo
  • Create equations from a script or external program
  • Take an equation from a browser or word processor and place in a symbolic algebra system
  • Autosave in standalone mode
  • Switch inline versus paragraph display mode
